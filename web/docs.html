AEGIS Engine primer

vault-native
An AEGIS vault is a programmable shell around Uniswap v4-style pool activity. The strategy controls how the vault borrows, repays, places liquidity, and reacts when market flow changes its inventory.
Debt is tracked as AEGIS L-unit principal rL. The borrow index turns principal into live liability. High LTV, weak collateral floor, or excess delta can trigger repairs or penalties.
  • L-unit borrow/repay: `BorrowL` creates rL debt and usable inventory; `RepayL` reduces liability and LTV.
  • Collateral floor and LTV: the runner marks vault equity after debt, computes collateral floor, and enforces max and hard LTV thresholds.
  • CL/LO attachment: concentrated-liquidity positions and limit-order positions belong to the vault and can be collected, resized, burned, withdrawn, or detached.
  • DFM BaseHook fees: dynamic fee logic affects the cost of swaps and the fees concentrated liquidity earns; surge state is visible in public pool fields.
  • Repairs: if the vault drifts too far, repairing can mean burning liquidity, swapping inventory, repaying, or suffering keeper-style repair costs.

Strategy lifecycle

one Python class

Your file defines `class Strategy`. The runner calls `on_start(state)` once, then calls `on_step(state)` for each public simulation step. Return a list of action objects. Keep it deterministic: no external calls, no hidden data, no private files, and no random behavior unless it is seeded by public state.

from aegis_challenge.api import BorrowL, MintRange

class Strategy:
    def on_start(self, state):
        self.done = False

    def on_step(self, state):
        if self.done:
            return []
        spacing = state.pool.tick_spacing
        mid = (state.pool.tick // spacing) * spacing
        self.done = True
        return [
            BorrowL(amount_l=10000),
            MintRange(lower_tick=mid - 12 * spacing, upper_tick=mid + 12 * spacing, liquidity=3000),
        ]

Borrow and repair

BorrowL(amount_l)

borrow

Purpose Creates AEGIS L-unit principal rL so the vault can fund ETH/USDC liquidity or order placement while tracking live debt.

  • Fields: `amount_l` integer principal to borrow.
  • When to use: before minting CL or LO positions that need ETH/USDC inventory, after checking debt headroom.
  • Use first: `state.vault.ltv_pips`, `state.vault.debt_l`, `state.vault.collateral_floor_l`, `state.eth_price`.
  • Risk: too much debt raises LTV and borrow cost; common errors include utilization cap, max LTV, or zero collateral floor.
from aegis_challenge.api import BorrowL

actions = [BorrowL(amount_l=12000)]

RepayL(amount_l | "all")

repair

Purpose Reduces AEGIS L-unit debt and lowers LTV when liability or inventory risk gets too high.

  • Fields: `amount_l` integer principal or `"all"`.
  • When to use: when LTV approaches repair bands, borrow cost dominates fee edge, or the vault exits risky inventory.
  • Use first: `state.vault.debt_l`, `state.vault.idle0`, `state.vault.idle1`, `state.vault.ltv_pips`.
  • Risk: repaying too early may starve liquidity; common errors include insufficient debt or insufficient idle balance.
from aegis_challenge.api import RepayL

actions = [RepayL(amount_l="all")]

SwapExactIn(token_in, amount_in, max_slippage_pips)

rebalance

Purpose Swaps vault inventory through the pool to repair ETH delta, create repay inventory, or rebalance idle balances.

  • Fields: `token_in` is `"token0"` for ETH or `"token1"` for USDC; `amount_in` is a Decimal; `max_slippage_pips` protects execution.
  • When to use: when ETH exposure breaches the strategy band or when repayment needs the opposite token.
  • Use first: `state.vault.delta`, `state.vault.idle0`, `state.vault.idle1`, `state.price`, `state.pool.dfm_total_fee_pips_estimate`.
  • Risk: swaps pay pool, LP, protocol, and DFM hook fees; common errors include slippage and insufficient idle balance.
from decimal import Decimal
from aegis_challenge.api import SwapExactIn

actions = [SwapExactIn(token_in="token1", amount_in=Decimal("2500"), max_slippage_pips=20000)]

Concentrated liquidity

MintRange(lower_tick, upper_tick, liquidity)

CL

Purpose Mints an ETH/USDC concentrated-liquidity position attached to the vault.

  • Fields: aligned lower tick, aligned upper tick, integer liquidity.
  • When to use: when you want fee exposure around the current ETH/USDC price and have idle inventory to deposit.
  • Use first: `state.pool.tick`, `state.pool.tick_spacing`, `state.pool.active_liquidity`, `state.vault.idle0`, `state.vault.idle1`.
  • Risk: tight ranges earn more when active but create inventory drift; common errors include invalid tick, insufficient idle, or max NFTs.
from aegis_challenge.api import MintRange

spacing = state.pool.tick_spacing
mid = (state.pool.tick // spacing) * spacing
actions = [MintRange(lower_tick=mid - 20 * spacing, upper_tick=mid + 20 * spacing, liquidity=5000)]

IncreaseRange(position_id, liquidity)

CL

Purpose Adds liquidity to an existing CL position when the range is working and inventory is available.

  • Fields: `position_id` from `state.positions`, additional `liquidity`.
  • When to use: when an existing range is active, fees are attractive, and delta/LTV are still within policy.
  • Use first: position tick range, uncollected fees, `state.vault.idle0`, `state.vault.idle1`, DFM fee state.
  • Risk: increasing too much concentrates inventory risk; common errors include missing position or insufficient idle balance.
from aegis_challenge.api import IncreaseRange

actions = []
if state.positions and state.vault.delta_normalized < 0.02:
    actions.append(IncreaseRange(position_id=state.positions[0].position_id, liquidity=1000))

DecreaseRange(position_id, liquidity)

CL

Purpose Pulls some liquidity out of a CL position without fully burning it.

  • Fields: `position_id` and liquidity to remove.
  • When to use: when a position is too inventory-heavy but you want to keep some fee exposure alive.
  • Use first: `state.positions`, `state.vault.delta_normalized`, `state.vault.ltv_pips`, fee accrual.
  • Risk: removing too much gives up fee capture; common errors include position not found or excessive liquidity requested.
from aegis_challenge.api import DecreaseRange

actions = []
if state.positions and state.vault.delta_normalized > 0.035:
    actions.append(DecreaseRange(position_id=state.positions[0].position_id, liquidity=1000))

CollectFees(position_id)

fees

Purpose Moves uncollected CL fees into idle vault balances so they can be redeployed or used for repair.

  • Fields: `position_id` from `state.positions`.
  • When to use: on a cadence or before repair so earned fees become usable idle balances.
  • Use first: `position.uncollected_fees0`, `position.uncollected_fees1`, `state.fees_earned_usd`.
  • Risk: collecting too often can add action costs; common errors include missing position.
from aegis_challenge.api import CollectFees

actions = []
for position in state.positions:
    if position.uncollected_fees1 > 0:
        actions.append(CollectFees(position_id=position.position_id))

BurnRange(position_id)

CL

Purpose Removes a CL position and returns its current inventory to the vault.

  • Fields: `position_id` from `state.positions`.
  • When to use: when a range is stale, risky, out of policy, or needed to unlock inventory for repayment.
  • Use first: `state.vault.ltv_pips`, `state.vault.delta_normalized`, range width, fee state.
  • Risk: burning can realize inventory drift; common errors include missing position or sequencing it before fee collection.
from aegis_challenge.api import BurnRange

actions = []
if state.positions and state.vault.ltv_pips > 970000:
    actions.append(BurnRange(position_id=state.positions[0].position_id))

Limit orders and advanced position control

PlaceLimitOrder(side, tick, liquidity)

LO

Purpose Places a single-sided Uniswap v4-style liquidity order at a tick to capture directional order flow.

  • Fields: `side` is `"sell0"` to sell ETH or `"sell1"` to sell USDC; aligned `tick`; integer `liquidity`.
  • When to use: when you want one-sided exposure to catch flow near the current tick without minting a two-sided range.
  • Use first: `state.pool.tick`, `state.pool.tick_spacing`, `state.limit_orders`, idle balances, recent fills.
  • Risk: crossed orders or stale ticks can fill immediately or fail; common errors include invalid tick, crossed limit order, insufficient idle, or max NFTs.
from aegis_challenge.api import PlaceLimitOrder

spacing = state.pool.tick_spacing
actions = [PlaceLimitOrder(side="sell0", tick=state.pool.tick + spacing, liquidity=500)]

CancelLimitOrder(order_id)

LO

Purpose Cancels an open limit order and returns unfilled inventory to idle balances.

  • Fields: `order_id` from `state.limit_orders`.
  • When to use: when an open order is too far from price, worsens delta, or should be replaced with a better tick.
  • Use first: order status, current tick, delta, LTV, and recent flow.
  • Risk: canceling too aggressively misses fills; common errors include order not found or already filled.
from aegis_challenge.api import CancelLimitOrder

actions = []
for order in state.limit_orders:
    if order.status == "open" and abs(order.tick - state.pool.tick) > 8 * state.pool.tick_spacing:
        actions.append(CancelLimitOrder(order_id=order.order_id))

WithdrawLimitOrder(order_id)

LO

Purpose Withdraws claimable proceeds from a filled limit order into the vault.

  • Fields: `order_id` from a filled `state.limit_orders` entry.
  • When to use: immediately after a fill or before any action that needs the claimable proceeds.
  • Use first: `order.status`, `order.claimable0`, `order.claimable1`, recent fills.
  • Risk: leaving filled orders unwithdrawn hides usable inventory; common errors include order not filled or order not found.
from aegis_challenge.api import WithdrawLimitOrder

actions = [
    WithdrawLimitOrder(order_id=order.order_id)
    for order in state.limit_orders
    if order.status == "filled"
]

DetachPosition(kind, id)

advanced

Purpose Detaches a CL or LO position from the vault when the challenge runner permits advanced position cleanup.

  • Fields: `kind` is `"CL"` or `"LO"`; `id` is the position or order id.
  • When to use: as an advanced cleanup path when the vault is unlocked and a position should leave active strategy control.
  • Use first: `state.positions`, `state.limit_orders`, repair events, and vault lock state.
  • Risk: detaching the wrong object removes strategy control; common errors include invalid kind, missing id, or unsafe locked state.
from aegis_challenge.api import DetachPosition

actions = []
if state.positions and state.vault.unlocked:
    actions.append(DetachPosition(kind="CL", id=state.positions[0].position_id))

Hidden-info and sandbox rules

fair competition

Your strategy can use

  • Public `state` fields passed into `on_start` and `on_step`.
  • Internal memory stored on `self` between steps.
  • Deterministic Python math and Decimal arithmetic.
  • Recent public swaps, fills, and repairs already emitted by the simulation.

Your strategy cannot use

  • No hidden fair price, hidden seeds, future flow, future trades, or ranked path parameters.
  • No private runner state, raw artifacts, local filesystem reads, network calls, unsafe imports, or side effects.
  • No nondeterministic replay behavior, long-running loops, or attempts to bypass action limits.

Scoring and robustness

USD first

Leaderboard score is net USD profit above the 100,000 USDC start, after costs and penalties. APR is annualized from elapsed simulated days as the run progresses, not fixed at the final 180-day horizon.

$ProfitEquity after debt minus the 100,000 USDC start, including fees, LO edge, inventory mark, borrow cost, repairs, and action costs.
DDeltaETH exposure is `d(vault_equity_after_debt) / d(price)`. Too much exposure means your run is betting on ETH.
FFeesDFM BaseHook fee logic affects swap costs and CL fees earned. Surge windows can make fee capture attractive but repairs more expensive.
RRanked robustnessRanked submissions should report median profit, worst decile, drawdown, delta-band time, and score variance across hidden seeded paths.

Starter delta-neutral vault

Borrow a moderate amount, mint a centered range, collect fees, and repair when delta or LTV drifts. Works as a first complete strategy; fails when the range is stale for too long.

from decimal import Decimal
from aegis_challenge.api import BorrowL, MintRange, CollectFees, RepayL

class Strategy:
    def on_start(self, state):
        self.done = False

    def on_step(self, state):
        if state.positions and state.step % 24 == 0:
            return [CollectFees(position_id=state.positions[0].position_id)]
        if state.vault.ltv_pips > 970000:
            return [RepayL(amount_l="all")]
        if self.done:
            return []
        spacing = state.pool.tick_spacing
        mid = (state.pool.tick // spacing) * spacing
        self.done = True
        return [BorrowL(amount_l=12000), MintRange(mid - 20 * spacing, mid + 20 * spacing, 5000)]

Limit-order rebalancer

Place orders just outside the current tick and withdraw filled orders. Works when flow clusters; fails if orders are too stale or crossed.

from aegis_challenge.api import BorrowL, PlaceLimitOrder, WithdrawLimitOrder

class Strategy:
    def on_start(self, state):
        self.bootstrapped = False

    def on_step(self, state):
        actions = [WithdrawLimitOrder(o.order_id) for o in state.limit_orders if o.status == "filled"]
        if not self.bootstrapped:
            spacing = state.pool.tick_spacing
            actions += [BorrowL(3000), PlaceLimitOrder("sell0", state.pool.tick + spacing, 500)]
            self.bootstrapped = True
        return actions
Delta repair bot

Swap only when exposure breaches a band. Works as a guardrail layer; fails when slippage and DFM fees are ignored.

from decimal import Decimal
from aegis_challenge.api import SwapExactIn

class Strategy:

        pass

            return []
        if state.vault.delta > 0 and state.vault.idle0 > 0:

DFM surge harvester

Watch DFM surge state and keep active liquidity near the price when the fee opportunity is worth the extra inventory risk. Works during surge windows; fails when repair swaps pay more DFM fee than the range earns.

from aegis_challenge.api import BorrowL, MintRange

Robustness-first ranked strategy

Use conservative borrow, wide ranges, cool-downs, and hard repair exits. Works across hidden paths; fails if it leaves too much capital idle.

from aegis_challenge.api import BorrowL, BurnRange, MintRange, RepayL

class Strategy:
    def on_start(self, state):
        self.placed = False

    def on_step(self, state):
        if state.vault.ltv_pips > 960000 or state.vault.delta_normalized > 0.04:
            actions = []
            if state.positions:
                actions.append(BurnRange(state.positions[0].position_id))
            actions.append(RepayL("all"))
            return actions
        if self.placed:
            return []
        spacing = state.pool.tick_spacing
        mid = (state.pool.tick // spacing) * spacing
        self.placed = True
        return [BorrowL(6000), MintRange(mid - 40 * spacing, mid + 40 * spacing, 1800)]

Debugging common failures

fix fast
Invalid tick

Round to `(state.pool.tick // state.pool.tick_spacing) * state.pool.tick_spacing`, then add or subtract multiples of spacing.

Insufficient idle balance

Collect fees, withdraw filled limit orders, decrease liquidity, or reduce requested liquidity before minting, increasing, or swapping.

Max LTV or hard LTV

Borrow less, repay earlier, burn a position, or widen the range so inventory drift does not force repair.

Slippage

Check DFM surge and active liquidity before swapping. Use `max_slippage_pips` deliberately, not as a magic bypass.

Sandbox violation

Remove filesystem, network, subprocess, unsafe imports, and nondeterministic calls. Keep logic inside `Strategy` and public `state`.

Bad ranked robustness

Test random practice paths. Favor median profit, low worst-decile loss, low drawdown, and high delta-band time.
